Записи пользователя — полный пересказ мыслей, идей и требований
=================================================================

Файл собран автоматически на основании всех сообщений пользователя во время сессии.
Цель: зафиксировать все мысли, предложения и требования пользователя так, как они были выражены,
чтобы последующие агенты и разработчики имели полный контекст.

1) Общая цель и ожидания
---------------------------------
- Репозиторий `libdrome` — небольшая JS‑библиотека/демо для тайлового мира и сущностей. Пользователь ожидает, что ИИ:
  - изучит репозиторий, подробно опишет содержимое файлов и архитектуру;
  - создаст ясную, долговременную документацию/контекст для последующих ИИ;
  - будет действовать строго по правилам, которые сам пользователь задаёт в `AI_CONTEXT/README.md`.

2) Работа с папкой `lib` и `liblist`
-----------------------------------
- Пользователь изначально просил переместить папку `lib` в корень и переименовать её так, чтобы любой ИИ первым делом изучал именно её (цель — сделать папку первоочередным местом для анализа).
- Правило: папка `lib/` должна содержать только JS‑файлы — модули и исполнители.
  - Модули: содержат только декларативные конструкции (классы, функции, методы, шаблоны). Модули не должны выполнять сайд‑эффекты при импорте.
  - Исполнители: содержат код запуска/инициализации/взаимодействия (всё, что не декларация).
- `liblist` должен содержать ровно пути к файлам (по одному на строку) — используется `index.html` для динамической загрузки модулей.

3) Создание AI_CONTEXT и разделение ролей README / SUMMARY
----------------------------------------------------------
- Пользователь потребовал создать отдельную корневую папку с контекстом для ИИ и сделать там:
  - `AI_CONTEXT/README.md` — статичный свод правил для генагентов (редактировать осторожно);
  - `AI_CONTEXT/SUMMARY.md` — динамичный файл с подробностями по коду, который агенты могут часто править.
- Правило: при добавлении записи в `AI_CONTEXT/CHANGELOG.md` необходимо обязательно актуализировать `AI_CONTEXT/SUMMARY.md`.
- Генерация контекста и саммари — отдельные обязанности: `README.md` — политики и ритуалы; `SUMMARY.md` — технические детали и описания файлов.

4) Концепция генагента (genagent) и ритуал итераций
--------------------------------------------------
- Введено понятие «генагент» — именованный агент, уполномоченный фиксировать основные изменения в репозитории.
- Правило возникновения генагента: генагентом считается агент, который создаёт новую запись в `AI_CONTEXT/CHANGELOG.md`.
- Правило именования: агент, вносящий запись в чейнджлог, может присвоить себе порядковое имя (например: "третий генагент") без дополнительного разрешения.
- Ритуал: перед каждой итерацией генагент обязан прочитать `AI_CONTEXT/README.md` и уведомить пользователя о прочтении; уведомление должно быть кратким и содержать порядковое имя генагента (например: "второй генагент").
- Правило: `AI_CONTEXT/README.md` не перечисляет имён генагентов — для этого служит `AI_CONTEXT/CHANGELOG.md`.

5) Политики редактирования и персонализации
-------------------------------------------
- Запись в чейнджлог должна содержать минимум: ISO‑дату, имя генагента, краткое описание изменений и список изменённых файлов.
- Генагентам разрешено оставлять личностные записи в `AI_CONTEXT/SUMMARY.md` и `AI_CONTEXT/CHANGELOG.md`.
- Генагентам разрешено создавать новые файлы в `AI_CONTEXT/` при необходимости документирования или фиксации изменений.
- Строгое правило: не выполнять git‑коммит без явного разрешения пользователя.
