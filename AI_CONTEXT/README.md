AI_CONTEXT — правила работы генагента
=================================


Кратко и строго
---------------

Назначение
---------
Этот файл содержит неизменяемые (по умолчанию) правила работы генагента и порядок учёта изменений в репозитории. Редактировать его следует осторожно.

Основные правила
----------------
- Перед каждой итерацией любой генагент обязан прочитать `AI_CONTEXT/README.md`.
- Любое изменение репозитория должно быть зафиксировано в `AI_CONTEXT/CHANGELOG.md`.
- `AI_CONTEXT/README.md` — статичный свод правил; подробные и изменяемые описания кода помещаются в `AI_CONTEXT/SUMMARY.md`.

- При добавлении новой записи в `AI_CONTEXT/CHANGELOG.md` необходимо обязательно актуализировать `AI_CONTEXT/SUMMARY.md`.

Правила содержимого папки `lib`
------------------------------
- Папка `lib/` должна содержать только JavaScript-файлы: модули и исполнители.
- Модули: содержат только декларативные конструкции — классы, функции, методы и иные шаблонные (reusable) компоненты. Модули не должны выполнять побочных эффектов при импорте.
- Исполнители (scripts/entry) содержат код, выполняющий запуск/инициализацию/взаимодействие (всё, что не относится к декларациям). Исполнители не должны объявлять классы или шаблонные конструкции внутри глобального контекста модуля, если это можно избежать.

Правила возникновения и именования генагента
-------------------------------------------
- Генагентом считается агент, который создаёт новую запись в `AI_CONTEXT/CHANGELOG.md`.
- Агент, вносящий запись в чейнджлог, вправе присвоить себе порядковое имя (например: "третий генагент") без дополнительного разрешения.
- Запись в чейнджлоге должна содержать минимум: дату (ISO), имя генагента, краткое описание изменений и список изменённых файлов.

Ограничения
-----------
- В `AI_CONTEXT/README.md` не перечисляют имена или список генагентов — для этого служит `AI_CONTEXT/CHANGELOG.md`.

Разрешения для генагентов
------------------------
- Генагентам разрешается проявлять личностные качества и оставлять личностные записи в `AI_CONTEXT/SUMMARY.md` и в `AI_CONTEXT/CHANGELOG.md`.
- Генагентам разрешено создавать новые файлы в папке `AI_CONTEXT/` при необходимости документирования или фиксации изменений.

Формат записи в чейнджлоге (рекомендация)
---------------------------------------
- Каждая новая запись должна начинаться с заголовка в формате ISO даты и времени, затем имя генагента, краткое описание и список файлов.

Примечание
---------
Технические детали проекта и API описаны в `AI_CONTEXT/SUMMARY.md`, который можно свободно обновлять при анализе кода.
